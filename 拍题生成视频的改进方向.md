# 拍题生成讲解视频改进方向（融合升级版）

更新时间：2026-04-15

本文融合两版计划书，并按当前代码现状重排为“可直接实施”的优化逻辑。
核心目标不是补单点功能，而是建立稳定主线：

题型先行 -> IR 结构化 -> 动作先校验 -> 再执行 -> 结果可回流

---

## 一、现状重定义（不是从零开始）

当前项目已具备基础能力：

* 已有 Geometry IR + Teaching IR 双层结构
* 已有步骤级场景更新和动作映射
* 已有折叠题基础动作链路
* 已有辅助线策略雏形
* 已有模板化 codegen 和静态校验

因此主问题已从“能不能动”变为“能不能稳定、可控、可迭代”。

你关心的5点结论如下：

1. 静态贴图模式：部分仍在，但已具备步骤化动画基础。
2. 步骤级动作脚本：已有 Teaching IR，但上游仍弱约束。
3. 折叠题专用模板：有基础处理，无模板族主线。
4. 辅助线策略引擎：有规则雏形，无完整决策闭环。
5. 双层 IR：已落地，当前应做升级而非重建。

---

## 二、优化逻辑（新版主线）

当前链路：

题图识别 -> 文本脚本 -> 动画侧反推动作

升级为：

题图识别 -> 题型识别层 -> Geometry IR -> Teaching IR -> 动作可执行性校验 -> 动画执行 -> 案例回流

新增三大核心模块：

1. 题型识别层（Problem Pattern Layer）
2. 动作可执行性校验器（Action Executability Checker）
3. 可学习数据闭环（Case Replay & Improvement Loop）

---

## 三、模块设计（与现有实现对齐）

### 1) 题型识别层

目的：先确定演示范式，再决定动作模板、辅助线策略和讲解节奏。

建议一级题型：

* static_proof
* metric_computation
* fold_transform
* dynamic_point
* similarity_congruence
* circle_geometry
* solid_geometry_section

建议输出：

```json
{
  "pattern_version": "v1",
  "problem_pattern": "fold_transform",
  "sub_pattern": "fold_with_perpendicular_construction",
  "presentation_mode": "transformation_first",
  "reasoning_mode": "invariant_then_auxiliary",
  "confidence": 0.92
}
```

### 2) 动作可执行性校验器

目的：在进入 codegen/scene 执行前，先做动作依赖和时序检查。

重点校验：

* 依赖对象是否存在
* 创建-使用顺序是否合法
* 本步 action 是否可在当前 scene 状态执行
* 是否存在可自动修补的问题

建议输出：

```json
{
  "checker_version": "v1",
  "status": "needs_repair",
  "issues": [
    {
      "type": "missing_dependency",
      "step_id": 3,
      "action_index": 1,
      "message": "Point H is referenced before creation"
    }
  ],
  "repair_suggestions": []
}
```

### 3) 案例回流闭环

目的：把一次生成的中间产物和结果沉淀为可检索案例，支撑持续迭代。

建议记录：

* 问题文本摘要
* 题型识别结果
* Geometry IR / Teaching IR
* 执行校验结果
* codegen 模式与渲染状态

---

## 四、IR 升级方向（v2）

### Geometry IR

在现有字段上增加：

* problem_pattern / sub_pattern
* goals（求解目标）

### Teaching IR

在现有字段上增加：

* problem_pattern / sub_pattern
* action 的依赖/修补友好字段

说明：

* 保持兼容 v1，采用“增量字段”方式升级。

---

## 五、折叠题模板库与辅助线引擎（主线化）

### 折叠题模板族

建议保留并主线化：

* fold_point_to_point_distance
* fold_with_perpendicular_construction
* fold_then_similarity
* fold_then_area

每个模板应声明：

* 必选动作序列
* 必校验不变量
* 推荐辅助线候选
* 动作依赖与失败修补策略

### 辅助线策略引擎

沿三段式升级：

1. 候选生成
2. 规则评分
3. 执行写回 + 校验联动

---

## 六、分阶段实施（含本轮立即实施项）

### 阶段A：题型识别层接入主链路

目标：先确定演示范式。

产出：

* ProblemPatternClassifier
* pattern 结果写入 metadata + geometry_ir

### 阶段B：动作可执行性校验器接入主链路

目标：Teaching IR 先校验再执行。

产出：

* ActionExecutabilityChecker
* 校验报告写入 metadata + debug
* 轻量自动修补（可安全修补项）

### 阶段C：案例回流基础设施

目标：中间产物可回放、可统计。

产出：

* CaseReplayRecorder
* 每次执行自动落盘 case record

### 阶段D：折叠题模板族增强

### 阶段E：辅助线策略引擎升级

### 阶段F：渲染表现增强（轨迹、标注、避让）

---

## 七、本轮实施边界（本次对话即开始）

本轮直接实施 A + B + C 的最小可用版本：

1. 增加题型识别层并接入 AnimationAgent.process
2. 增加动作可执行性校验器并在 Teaching IR 后执行
3. 增加案例记录器并自动落盘
4. 补充对应测试用例

暂不在本轮完成：

* 折叠题模板族完整目录化
* 辅助线评分器复杂模型
* 渲染侧高级轨迹/镜头系统

---

## 八、验收标准（本轮）

本轮完成后，单题生成流程需满足：

1. metadata 中存在 problem_pattern 信息
2. metadata 中存在 teaching_ir_execution_check 报告
3. output/debug 中存在 case record 文件
4. 新增测试通过且不破坏现有测试

---

## 九、结论

当前系统已过“功能补洞期”，进入“结构升级期”。
本轮优先落地“题型识别 + 动作校验 + 案例回流”三件事，
把现有 IR 能力从“可用”升级为“可控、可复盘、可迭代”的主线系统。
